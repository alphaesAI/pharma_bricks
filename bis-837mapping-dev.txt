BIZTALK MAP FILES (.btm)
Purpose: Define transformations from X12 XML to flat file format
------------------------------------------------------------------------------

RegularProfClaimPatientMap.btm - BizTalk map for PATIENT/DEPENDENT claims:
  * Maps 837 Professional X12 XML loops for dependent patients
  * Source: X12_00501_837_P.xsd (X12 XML structure)
  * Target: RegularProfClaimSchema.xsd (flat file format)
  * Handles loops: 2000B (Subscriber), 2000C (Patient), 2300 (Claim), 2400 (Service)

RegularProfClaimPatientMap.btm.cs - Auto-generated C# code for PatientMap

RegularProfClaimSubscriberMap.btm - BizTalk map for SUBSCRIBER claims:
  * Maps 837 Professional X12 XML loops for subscriber (non-dependent) claims
  * Source: X12_00501_837_P.xsd
  * Target: RegularProfClaimSchema.xsd
  * Handles loops: 2000A (Billing Provider), 2000B (Subscriber), 2300, 2400

RegularProfClaimSubscriberMap.btm.cs - Auto-generated C# code for SubscriberMap

------------------------------------------------------------------------------

Purpose: Define flat file output structure for Databricks ingestion

RegularProfClaimSchema.xsd - Output flat file schema (non-repriced claims):
  * Defines fixed-width/positional output fields
  * Field definitions: Header, Subscriber, Patient, Claim, Service line items
  * Position/length specifications for each field

RegularProfClaimSchema.xsd.cs - Auto-generated C# class for schema

RepriceProfClaimSchema.xsd - Output schema for repriced/adjusted claims:
  * Additional fields for claim repricing information

RepriceProfClaimSchema.xsd.cs - Auto-generated C# class for repriced schema

RulesProfClaimSchema.xsd - Output schema for claims with business rules applied

RulesProfClaimSchema.xsd.cs - Auto-generated C# class for rules schema

------------------------------------------------------------------------------
INPUT SCHEMA FILES (.xsd)
Purpose: Define X12 837 Professional EDI XML structure
------------------------------------------------------------------------------

Schema/X12_00501_837_P.xsd - X12 837 Professional schema (main version):
  * Defines ISA, GS, ST envelopes
  * 1000A (Submitter), 1000B (Receiver)
  * 2000A (Billing Provider HL loop)
  * 2000B (Subscriber HL loop)  
  * 2000C (Patient HL loop)
  * 2300 (Claim information)
  * 2400 (Service line details)

X12_00501_837_P_Single.xsd.cs - Auto-generated C# class for single claim schema

================================================================================
WORKFLOW (How this integrates with Databricks)
================================================================================

1. RAW X12 EDI comes from payer/health plan
2. BizTalk receives EDI via receive port
3. BizTalk parses EDI using X12_00501_837_P.xsd schema
4. BizTalk executes maps (PatientMap or SubscriberMap based on claim type)
5. Output: Fixed-width flat files per RegularProfClaimSchema.xsd
6. Flat files land in Azure Blob/S3 storage
7. Databricks (bis-databricks-dev) picks up files for CSV conversion

================================================================================
KEY INSIGHT FOR REBUILD
================================================================================

This BizTalk layer can be REPLACED by Python X12 parser:
- pyx12 library parses X12 EDI → XML/JSON
- Custom mapper converts to Databricks-compatible format
- Skip fixed-width intermediate step
- Go directly from X12 → Delta tables
